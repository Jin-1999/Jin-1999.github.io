    # 什么是闭包？

    ​ 闭包指的是那些引用了另一个函数作用域中变量的函数，通常是在嵌套函数中实现的。

      举个简单的🌰

    function f1() {
    let num = 9;
    function f2() {
        console.log(num)
    }
    return f2
}

let result = f1();  //9

      正常来说，在函数（f1）外部，我们是无法拿到num的值，父对象访问不到子对象的变量。但是闭包使之变成可能。

      对f1和f2来说，f2可以访问f1内部的所有变量，反之则不行。对f1和f1外部的全局对象来说，f1可以访问外部的全局变量，反之则不行。

      在JS中，子对象会逐级向上寻找父对象的变量，及父对象的所有变量，对子对象都是可见的，反之则不成立。

    # 闭包的使用

      闭包可以让我们访问另一个函数内部的变量，那么闭包在哪些地方可以使用呢？下面举几个🌰

    # 防抖

    function debounce(fn, wait, isImmediate) {
    wait = wait === undefined ? 300 : wait;  //默认300ms
    var timer, result;
    return function () {
        var context = this;
        var args = arguments;
        if (timer) {
            clearTimeout(timer);
        }
        if (isImmediate) {
            if (!timer) result = fn.apply(context, args);
            timer = setTimeout(function () {
                timer = null;
            }, wait);
        } else {
            timer = setTimeout(function () {
                fn.apply(context, args);
            }, wait);
        }
        return result
    }
}

    # 定时器传参

      使用闭包，可以使setTimeout(function(){}, 100)的第一个参数函数带上参数。以此可以类推其余相似的函数。

    let fn1 = (a) => {
    let fn2 = () => { console.log(a) }
    return fn2
}
let fun = fn1(100)
setTimeout(fun, 500)   //100

    # 闭包的用途

      从上面几个例子可以看出，闭包的主要可以

    • 从外部读取函数内部的变量
    • 将变量始终保存在内存中

    # 闭包的坏处

      因为闭包会保留它们包含函数的作用域，所以比其他函数更占用内存。过度使用闭 包可能导致内存过度占用，因此建议仅在十分必要时使用。V8 等优化的 JavaScript 引擎会 努力回收被闭包困住的内存，不过我们还是建议在使用闭包时要谨慎。
